# P0-A core demo script

Мета демонстрації — один стабільний наскрізний сценарій українською: Brain Dump → AI аналіз → preview → apply → Today → edit → Undo.

## Підготовка

- Відкрийте production Vercel URL на мобільній ширині 390 px.
- Переконайтесь, що auth test account має доступ до Gateway/PocketBase і чистий demo workspace.
- Перевірте, що існує Google event «Командний синк» 11:00–11:45.

## Контрольований Brain Dump

Вставте або продиктуйте:

> Мені треба підготувати перший випуск подкасту, написати Марії про запис, ще не забути замовити корм коту. Думаю зробити окремий епізод про синдром самозванця, але не знаю, чи варто зараз ще запускати YouTube. У четвер об 11 у мене командний синк.

Очікування: AI класифікує задачі, ідеї та контекст, показує confidence/рекомендацію і не записує зміни без підтвердження. Якщо confidence низький, ставиться не більше 1–2 критичних питань.

## Demo path

1. `Brain Dump` → **Написати текстом** або **Диктувати**.
2. Якщо браузер не дає microphone permission, оберіть **Написати текстом** — це штатний fallback.
3. Збережіть чернетку і дочекайтесь **Аналіз готовий**.
4. Відкрийте **Переглянути пропозиції**. Покажіть tasks, ideas, context і зайнятий Google слот.
5. Натисніть **Застосувати план**.
6. На **Сьогодні** покажіть deterministic time placement, locked event і наступну задачу.
7. Відкрийте task detail → **Редагувати**, змініть назву, збережіть і переконайтесь, що зміна видима.
8. Виконайте задачу. У snackbar натисніть **Скасувати** (Undo) і переконайтесь, що задача повернулась у попередній стан.
9. Перезавантажте сторінку: persisted Today має залишитися, а не повернутись до локального mock.

Не демонструйте ручне видалення даних, необроблені server errors, secrets або непідключені інтеграції як готові функції.
